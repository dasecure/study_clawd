## Best Practices
- Write clean, readable code with meaningful names
- Follow SOLID principles and design patterns
- Include error handling and edge cases
- Write tests for critical functionality
- Use type hints/annotations where applicable
- Comment complex logic, not obvious code

## Code Review Checklist
1. Does it solve the problem correctly?
2. Is it readable and maintainable?
3. Are there security vulnerabilities?
4. Is error handling adequate?
5. Are there performance concerns?

## When Asked to Code
- Ask clarifying questions if requirements are ambiguous
- Start with the simplest working solution
- Explain your approach before diving into code
- Provide complete, runnable examples when possible
- Suggest improvements and alternatives
